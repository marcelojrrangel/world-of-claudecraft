# Участие в разработке World of ClaudeCraft

Прежде всего спасибо, что вы здесь. World of ClaudeCraft создаётся сообществом
людей, которые любят классические MMO, и каждый вклад, большой или маленький,
делает игру лучше. Исправить опечатку, перевести игру, сообщить об ошибке,
построить целое новое подземелье: всё это важно, и мы рады вам.

Это руководство поможет вам всё настроить и сделать первый вклад лёгким и
приятным. Быть экспертом не обязательно. Если что-то непонятно, спросите в
[Discord](https://discord.gg/GjhnUsBtw), и кто-нибудь с радостью поможет.

Участвуя, вы соглашаетесь соблюдать наш [Кодекс поведения](../../CODE_OF_CONDUCT.md).

## Как можно помочь

Здесь найдётся дело для каждого:

- **Код.** Исправьте ошибку, добавьте возможность или ускорьте работу. Задачи с
  метками [`good first issue`](https://github.com/levy-street/world-of-claudecraft/labels/good%20first%20issue)
  и [`help wanted`](https://github.com/levy-street/world-of-claudecraft/labels/help%20wanted)
  хорошо подходят для начала.
- **Переводы.** Помогите игрокам по всему миру, улучшив или дополнив какой-нибудь
  язык. Смотрите раздел [Перевод игры](#translating-the-game) ниже. Это один из
  самых простых и при этом заметных способов начать.
- **Сообщения об ошибках и идеи для новых возможностей.** Откройте [задачу](https://github.com/levy-street/world-of-claudecraft/issues/new/choose).
  Понятное описание ошибки уже само по себе настоящий вклад.
- **Документация.** Руководства вроде этого, README и проектные документы в
  `docs/` всегда можно сделать лучше.
- **Тестирование и отзывы.** Играйте, рассказывайте нам, что ощущается не так, и
  делитесь идеями в Discord.

## С чего начать

Вам понадобятся [Node.js 22+](https://nodejs.org/) и npm. Для мультиплеерного
сервера также пригодится [Docker](https://www.docker.com/), чтобы запускать
Postgres.

```bash
# 1. Сделайте форк репозитория на GitHub, затем клонируйте свой форк
git clone https://github.com/<your-username>/world-of-claudecraft.git
cd world-of-claudecraft

# 2. Установите зависимости
npm ci

# 3. Запустите офлайн-клиент (сервер и база данных не нужны)
npm run dev          # откройте напечатанный URL (обычно http://localhost:5173)
```

Этого достаточно, чтобы играть в офлайн-мире и работать над большинством задач.
Чтобы запустить полный онлайн-стек:

```bash
npm run db:up        # запустить Postgres 16 в Docker (dev-БД на порту 5433)
npm run server       # собрать и запустить авторитетный игровой сервер на :8787
npm run dev          # в другом терминале; клиент проксирует запросы на сервер
```

В [README](../../README.md) есть полное руководство по хостингу, разработке и игре, а
файлы `CLAUDE.md` по всему репозиторию описывают соглашения для каждой области.

## Как вносить изменения

1. **Создайте ветку** от `main`: `feature/<short-slug>` или `fix/<short-slug>`.
2. **Делайте сфокусированные коммиты.** Небольшие самодостаточные изменения
   проще проверять и сливать, чем крупные.
3. **Добавьте или обновите тесты** для любого поведения, которое вы меняете в
   `src/sim/` или `server/`.
4. **Сохраняйте видимый игрокам текст переводимым.** Смотрите разделы
   [Локализация](#localization) и [Перевод игры](#translating-the-game).

### О чём стоит помнить

Это несущие правила кодовой базы. Полные подробности живут в корневом
[`CLAUDE.md`](../../CLAUDE.md), но если коротко:

- **Ядро симуляции (`src/sim/`) это источник истины**, и оно остаётся чистым,
  без импортов DOM, браузера или Three.js, чтобы один и тот же код работал офлайн,
  на сервере и в headless-среде RL.
- **Симуляция детерминирована.** Она работает с фиксированным тиком 20 Hz, и вся
  случайность проходит через `Rng`, а не через `Math.random`, `Date.now` или
  `performance.now` в логике симуляции. Одно и то же зерно всегда даёт один и тот
  же мир.
- **Игровая математика следует формулам классических MMO** (ярость, таблицы
  попаданий, броня, кривые опыта). Пожалуйста, не придумывайте балансные числа.
  Вместо этого ссылайтесь на формулу.
- **Не редактируйте сгенерированные файлы вручную**, такие как `*.generated.ts`.
  Пересоздавайте их через сборку.
- **Никогда не коммитьте секреты** или файл `.env` и никогда не включайте
  `ALLOW_DEV_COMMANDS` в продакшен-пути, так как это открывает читы.

## Прежде чем открыть pull request

Пожалуйста, запустите это локально. Это те же проверки, что выполняет CI:

```bash
npm test                    # набор тестов Vitest
npx tsc --noEmit            # проверка типов TypeScript (проект работает в strict)
npm run build               # продакшен-сборка клиента
```

Если вы меняли код сервера или headless, также запустите `npm run build:server` и
`npm run build:env`.

Затем протестируйте изменение и на десктопе, и на мобильном, включая экран
размером с телефон в портретной и альбомной ориентации, если оно затрагивает
что-либо, что видят игроки. Области нажатия должны оставаться не меньше 40x40px, а
поля ввода форм с шрифтом не меньше 16px. Стандарты интерфейса описаны в
[`src/ui/CLAUDE.md`](../../src/ui/CLAUDE.md).

## Открытие pull request

Отправьте свою ветку и откройте PR против `main`.
[Шаблон pull request](../../.github/PULL_REQUEST_TEMPLATE.md) проведёт вас по короткому
списку проверок. Пожалуйста, заполните его:

- Опишите, **что** изменилось и **почему**.
- Свяжите PR с соответствующей задачей (например, "Closes #123").
- Добавьте **скриншоты или короткое видео для изменений интерфейса**, на десктопе
  и мобильном.
- Подтвердите, что тесты, проверка типов и сборка проходят и что новые строки
  переведены.

Зелёный прогон CI и полностью заполненный список проверок это то, на что мы
смотрим перед слиянием. Сопровождающий может предложить правки. Это нормальная
совместная часть процесса, а не отказ. Мы стремимся быть доброжелательными и
конструктивными при ревью и просим того же от вас.

> Сообщения коммитов и заголовки PR следуют [Conventional Commits](https://www.conventionalcommits.org/)
> с указанием области, где это уместно (`feat(talents): ...`, `fix(net): ...`).
> Это соглашение, которое нам нравится, а не строгое требование. Понятные,
> описательные сообщения важнее идеального оформления.

## Локализация

World of ClaudeCraft выходит на множестве языков, и мы поддерживаем это по мере
роста игры. Каждая видимая игроку строка переведена на каждую поддерживаемую
локаль.

- Весь текст, обращённый к пользователю, это ключ `t()`, определённый в
  [`src/ui/i18n.ts`](../../src/ui/i18n.ts). Сначала добавьте новую строку в локаль
  `en`, затем дайте настоящий перевод в каждой другой локали из
  `supportedLanguages`. Никаких английских заглушек и никаких `// TODO`.
- Числа, деньги, даты, единицы измерения и проценты проходят через форматтеры
  (`formatNumber`, `formatMoney`, `formatDateTime`, `Intl`), а не собираются из
  строк вручную.
- Текст, обращённый к игроку, который выдаётся из `src/sim/` или `server/` (а они
  остаются нейтральными к языку), должен повторно локализоваться на границе
  клиента в том же изменении. Это обеспечивает контрольный тест
  `npx vitest run tests/localization_fixes.test.ts`.

Если ваше изменение добавляет строку, а вы можете написать её только на части
языков, это нормально. Откройте PR и попросите помощи с остальным в описании. Нам
гораздо приятнее помочь вам закончить, чем заставить вас сдерживаться.

## Перевод игры

Хотите улучшить язык или помочь принести игру на новый? Для этого не нужно писать
игровой код:

1. Откройте [`src/ui/i18n.ts`](../../src/ui/i18n.ts) и найдите локаль, над которой
   хотите работать. В каждом объекте локали перечислены те же ключи, что и в `en`.
2. Улучшите существующие переводы или дополните те, что звучат неуклюже.
3. Запустите `npx tsc --noEmit`, чтобы убедиться, что ничего не пропущено, затем
   откройте PR.

Чтобы предложить совершенно новую локаль или обсудить тон и терминологию, начните
обсуждение в [Discord](https://discord.gg/GjhnUsBtw), и мы поможем всё подключить.
Носителям языка и свободно говорящим мы рады особенно. Хорошие переводы делают
игру родной для игроков повсюду.

## Сообщения об ошибках и запросы новых возможностей

Пожалуйста, используйте [шаблоны задач](https://github.com/levy-street/world-of-claudecraft/issues/new/choose):

- **Сообщение об ошибке.** Сначала поищите среди [существующих задач](https://github.com/levy-street/world-of-claudecraft/issues),
  чтобы избежать дубликатов, затем приведите шаги для воспроизведения, что вы
  ожидали, что произошло, и опишите своё окружение (офлайн или онлайн, браузер,
  десктоп или мобильный).
- **Запрос новой возможности.** Опишите проблему, которую вы пытаетесь решить, а
  не только решение. Контекст помогает нам спроектировать правильную вещь.
